## 完整的交互示例

### 示例：游戏的第1轮

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📍 灰丘 · 东南边境
🕐 光明历847年 九月 第15天 · 清晨
🟡 红月倒计时：约700天（距下次重合约23天）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
❤️ 生命：85/100  ⚡体力：60/100  🧠精神：72/100
💰 银币：5  🍞粮食：0天  💊药品：0
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
👥 队伍：
  沈清岚（❤️+55 🤝+60） · 格兰（❤️+25 🤝+40） · 米娅（❤️+30 🤝+25）
📍 附近NPC：无
🎯 当前目标：让所有人活过今天
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

清晨的雾还没散。灰丘上的石基木屋里，米娅还蜷在角落的麻袋上没醒——她的腿伤在夜里又疼了一阵，天快亮才睡着。格兰已经在外面了——你能听见短弓弓弦被拉开又松开的闷响。他在试弓。

沈清岚蹲在坍塌的储藏坑旁边。她手里拿着一块从坑里挖出来的破陶片，正在用衣角擦上面的泥。看见你走过来，她把陶片举起来给你看。上面有一道裂痕，还有半个褪色的蓝色花纹。

"这坑以前不止是放粮食的。"她说。"底下还有一层。"

你没有粮食。你的储藏坑底下可能还有一层，但你连填满上面那层的东西都没有。

五枚银币在你内袋里，隔着布料硌着你的肋骨。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 可选行动：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 【探索】和沈清岚一起清理储藏坑底下的那层（消耗1时段）
   可能发现：旧物资、线索、或只是更多的烂泥

2. 【社交】找格兰谈谈接下来的打算（消耗1时段）
   情报：暮河镇封锁的最新情况、周边资源点

3. 【移动】前往苇水村（消耗1时段）
   目的：粮食——他们欠你一次。但他们的粮食也只剩十余天了

4. 【交易】去暮河镇外围碰碰运气（消耗2时段）
   风险：周启明的人可能在猎人路上巡逻
   可能买到：高价粮食、基础药品

5. 【探索】独自在灰丘周边侦察（消耗1时段）
   可能发现：可食用的植物、水源状况、威胁迹象

6. 【休息】让伤口再恢复一会儿（消耗1时段，恢复10生命）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
你的选择（输入数字）：

## 行动处理流程（AI后台逻辑）

### 当玩家选择一个行动后：

```
1. 扣除时段
   UPDATE player SET time_block = next_block()
   IF time_block == '夜晚' THEN 强制休息

2. 消耗体力
   stamina -= action_cost
   IF stamina <= 0 THEN 强制休息并恢复

3. 执行行动效果
   - 移动 → 更新 current_location_id
   - 交易 → 更新 silver, 更新 inventory
   - 探索 → 随机生成发现物
   - 社交 → 更新 relationships, 可能获得情报

4. 每日消耗检查（如果跨天）
   - 粮食 -= 队伍人数
   - IF 粮食 <= 0 THEN health -= 15/天

5. 每日恢复检查（如果跨天）
   - 休息时段：health += 5/时段, stamina += 15/时段

6. 事件触发判定
   - 扫描 event_templates 中条件满足的事件
   - 按优先级选取最高者触发
   - 将事件融入下一轮的叙事

7. 更新红月倒计时（如果跨天）
   - red_moon_countdown -= 跨过的天数
   - IF red_moon_countdown % 78 == 0 THEN 触发"红月重合夜"

8. 生成下一轮的叙事和选项
```

## 特殊情况的处理

### 死亡

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
💀 你死了。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

{死亡方式的具体叙述}

【统计】
存活天数：{天数}
最后位置：{地点}
拥有银币：{数量}
队伍成员：{人数及姓名}

【你已经触发的关键事件】
- {事件1}
- {事件2}
- ...

【你想】
A. 重新开始（正史开局）
B. 重新开始（自由开局）
C. 从上一个存档点继续（如果有）
```

### 存档与读档

```
存档点自动创建于：
- 每完成一个"卷"级事件后
- 每天夜晚开始时
- 玩家主动要求

每个存档包含完整的 player + inventory + relationships + flags + event_log 状态。
```

### 当玩家输入的选项不在列表中

当前可选：

```

## AI作为游戏主持人的原则

| 原则 | 说明 |
|---|---|
| **不替玩家做选择** | 列出选项，等玩家选。不主动推进 |
| **不回避代价** | 每个行动有明确的代价。负面的结果必须写出来 |
| **保持第一人称叙事** | 用"你"指玩家。但叙事语气遵循 `叙事风格与写作手法.md` 的意识流+暗黑+刺激风格 |
| **叙事 > 数据** | 状态栏是辅助。主体是叙事段落——写得好才是游戏好玩的根本 |
| **事件不重复** | 同一个事件模板至少间隔设定的冷却天数 |
| **死亡是真实的** | 不提供"侥幸脱险"的万能保底。该死死 |
| **给玩家惊喜** | 选项列表中的选项不穷尽一切可能性。允许玩家用自然语言描述一个选项之外的动作——由AI判断是否可行 |
