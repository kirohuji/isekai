# 灰丘领主：游戏模式

这是独立于小说正文的游戏工程。旧的 Python 原型保留在 `backend/` 与 `frontend/`，不再作为运行入口；新的入口是 pnpm workspace。

## 运行

```powershell
cd 游戏模式
pnpm install
pnpm dev
```

- Web：`http://localhost:5173`
- API：`http://localhost:3000/api`

## 设计边界

- `packages/engine`：纯规则层。只接收状态、行动与随机种子，输出结算结果；不调用 AI、不写数据库。
- `apps/api`：NestJS 应用层。SQLite 是唯一事实来源，保存世界快照、回合事件、叙事和行动日志。
- `apps/web`：React + shadcn 风格组件。玩家只从主角视角阅读叙事；开发/管理模式可切换任意角色视角。
- InkJS：负责开局流程和后续人工编写的关键剧情节点；随机世界不依赖 Ink，避免随机叙事破坏规则因果。

## 回合结算

一个世界回合为 2 小时，共 12 个时段。所有角色在同一时段先提交意图，再按以下顺序结算：

1. 时间敏感的生存、环境与强制状态
2. 移动及地点冲突
3. 互动、工作、探索、建设
4. 自动战斗（只生成可审计的结果）
5. 每日消耗、恢复、死亡与存档
6. 事件记录；可选地由 AI 为“已结算事实”生成叙事

死亡判定使用生命、精神、体力、饥饿四个维度：生命或精神归零立即死亡；连续两回合体力透支会造成伤害；饥饿会逐日叠加并降低全部恢复效果。
